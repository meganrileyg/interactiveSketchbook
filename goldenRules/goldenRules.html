8 Golden Rules: Reading Response

Which rule feels most absent from your micro-project page?

The Universal Usability rule feels the most absent in my micro-project, as there is only one real task for users to accomplish, and one way to do it. Both a novice and an expert run the risk of a randomized “drop call,” which experienced users may be more aware of, but are not safe from. Any user must go through the same system of key pressing to try to “answer” the phone. With this, there is no design within the system that is better suited for users with varying needs; it is all the same.

Which rule would most improve your sketchbook interface?

The sixth rule, Permitting Easy Reversal of Actions, would benefit my sketchbook interface. Including the menu consistently on the screen while the user navigates through projects and sketches would aid the user in easy backstepping through the system if they click on the wrong sketch.

Which rule is hardest to implement honestly?

Rule four, Designing Dialogues to Yield Closure, is the hardest to implement into my sketchbook interface honestly. Navigation through the sketchbook is in the user's control, though there is no real way to guide them through from beginning to end, unless perhaps the user is made to cycle through the sketchbook until they have passed all of the sketches. The micro-project has some dialogue that urges the user through beginning, middle, and end, but it is limited and can be interrupted by a dropped call.

Did any rule contradict how you designed your system originally?

Rule seven, Keep Users in Control, contradicts the original design of my micro-project the most. The user is in charge of pressing the correct key to get to an end result, but the system is designed to be frustrating. The randomized drop call feature interrupts the user's sense of control, and can make them frustrated if it interrupts their result too many times.
